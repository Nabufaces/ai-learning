# Learning Artifact Review

Use this skill when reviewing a roadmap week, lab, harness note, eval case, or agent workflow in this repository.

## Checklist

1. State the artifact path and intended learner.
2. Check that it has a concrete goal.
3. Check that it includes at least one runnable or inspectable task.
4. Check that the success criteria are observable.
5. Check whether a failure case or eval should be added.
6. Return findings first, then a short improvement summary.

## Review Priorities

- Missing runnable practice.
- Vague success criteria.
- Tool or agent behavior without failure handling.
- Harness concepts described without trace, budget, permission, or checkpoint implications.
- External facts without source links.
